# v1.4 更新日志

更新时间：2026-04-24

## 概览

v1.4 聚焦工程治理与 DDD 收敛，目标是让六层职责更清晰、领域模型更聚焦、查询入口更统一，为后续功能迭代和部署稳定性打基础。

---

## 核心更新

### 1) 六层职责进一步收敛

- 明确 `trigger -> api.application -> app -> domain -> infrastructure` 调用链
- `trigger` 仅承担协议转换与异常包装
- `app` 负责用例编排与 DTO 映射
- `domain` 负责业务规则与领域行为

---

### 2) 领域上下文拆分落地

`easy-agent-domain` 形成三个业务上下文：

- `cn.caliu.agent.domain.agent`
- `cn.caliu.agent.domain.user`
- `cn.caliu.agent.domain.session`

主要变化：

- 用户认证与订阅能力归位到 `domain.user`
- 会话绑定与创建能力归位到 `domain.session`
- `runtime` 继续并入 `domain.agent`，与 Agent 装配/发布流程保持一致

---

### 3) API 契约与 DTO 结构整理

- API DTO 按业务域重组：
- `api.dto.agent.config.*`
- `api.dto.agent.chat.*`
- `api.dto.user.auth.*`
- 新增 `api.application.*` 契约，明确应用层对触发层的服务边界

---

### 4) 查询入口统一（去冗余）

为避免“同类查询多套入口”的长期维护成本，v1.4 完成查询入口收敛：

- 保留统一分页查询：`POST /api/v1/agent_config_page_query`
- 保留业务特化查询：广场列表、我的订阅
- 移除冗余列表查询路径（旧 `list/my-list` 语义入口）
- 仓储层移除对应冗余方法：`queryList`、`queryMyList`

---

### 5) 领域模型增强与遗留模型清理

- 引入 `UserId` 值对象，统一用户 ID 的规范化与校验规则
- `AgentConfigPageResultVO` 语义升级为 `AgentConfigPageQueryResult`
- 删除遗留弃用模型：
- `AgentConfigManageVO`
- `UserAccountVO`
- `AgentSessionBindVO`

---

## 兼容性说明

- 前端若仍调用旧列表接口，请切换到 `agent_config_page_query`
- 本次无新增数据库表结构变更，沿用 v1.3 阶段已有脚本
- 订阅与用户能力继续依赖：
- `ai_user_account`
- `ai_agent_subscribe`
