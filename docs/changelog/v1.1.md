# v1.1 更新日志

更新时间：2026-04-19

## 概览

v1.1 围绕“Supervisor 多 Agent 动态协作 + 流式前端可视化”进行了增强，目标是让主 Agent 可以在一次会话中多轮调度子 Agent，并以标准 SSE 事件流输出思考过程和回复过程。

## 核心更新

### 1. 新增 Supervisor 编排能力

- 新增 `supervisor` 工作流类型
- 支持主 Agent（router）动态决定：
- `action=route`：继续分派给下一个子 Agent
- `action=final`：结束并输出最终答案
- 支持 `max-iterations` 控制最大路由轮次，避免无限循环

涉及代码：

- `easy-agent-domain/src/main/java/cn/caliu/agent/domain/agent/service/armory/node/workflow/SupervisorRoutingAgent.java`
- `easy-agent-domain/src/main/java/cn/caliu/agent/domain/agent/service/armory/node/workflow/SupervisorAgentNode.java`

### 2. 主 Agent / 子 Agent 流式输出标准化

新增统一 marker（内部事件标记）：

- `[[AGENT_THINKING]]`
- `[[AGENT_ROUTE]]`
- `[[AGENT_REPLY]]`
- `[[AGENT_FINAL]]`

触发层将其映射为前端可消费的标准字段：

- `type: thinking | route | reply | final`
- `agentName`
- `content`
- `routeTarget`
- `partial`
- `finalResponse`

涉及代码：

- `easy-agent-types/src/main/java/cn/caliu/agent/types/common/AgentStreamMarker.java`
- `easy-agent-trigger/src/main/java/cn/caliu/agent/trigger/http/AgentServiceController.java`
- `easy-agent-api/src/main/java/cn/caliu/agent/api/dto/ChatStreamEventResponseDTO.java`

### 3. Router 输出协议增强

Router 必须输出单个 JSON 对象（禁止附带额外文本），建议结构：

```json
{"thought":"简短思考","action":"route","nextAgent":"MathExpert","reply":"阶段性回复"}
```

或

```json
{"thought":"简短思考","action":"final","reply":"最终回复"}
```

新增 `thought` 字段用于思考流展示，`reply` 用于主 Agent 阶段输出或最终输出。

## YML 配置示例（Supervisor）

```yaml
ai:
  agent:
    config:
      tables:
        supervisorDemo:
          app-name: SupervisorDemoApp
          agent:
            agent-id: 100010
            agent-name: Supervisor Demo
            agent-desc: Supervisor routes to expert agents and returns final answer.
          module:
            ai-api:
              base-url: https://dashscope.aliyuncs.com/compatible-mode/
              api-key: ${DASHSCOPE_API_KEY}
              completions-path: v1/chat/completions
              embeddings-path: v1/embeddings
            chat-model:
              model: qwen-plus
            agents:
              - name: SupervisorRouter
                description: Decide next step and return strict JSON only.
                instruction: |
                  You are the supervisor router.
                  Output one JSON object only:
                  {"thought":"简短思考","action":"route","nextAgent":"MathExpert","reply":"阶段性回复"}
                  or
                  {"thought":"简短思考","action":"final","reply":"最终回复"}
              - name: MathExpert
                description: Solve math problems.
                instruction: |
                  你是数学专家，请输出准确且简洁的结果。
              - name: JavaExpert
                description: Provide Java/Spring backend code solutions.
                instruction: |
                  你是 Java/Spring 专家，请输出可运行代码与关键说明。
              - name: GeneralExpert
                description: Handle general questions.
                instruction: |
                  你是通用专家，请输出简洁建议。
            agent-workflows:
              - type: supervisor
                name: SupervisorPipeline
                description: Supervisor routes experts and synthesizes answer.
                router-agent: SupervisorRouter
                max-iterations: 6
                sub-agents:
                  - SupervisorRouter
                  - MathExpert
                  - JavaExpert
                  - GeneralExpert
            runner:
              agent-name: SupervisorPipeline
              plugin-name-list:
                - myTestPlugin
                - myLogPlugin
```

## chat_stream 事件流说明

接口：

- `POST /api/v1/chat_stream`
- `Content-Type: application/json`
- `Accept: text/event-stream`

SSE 示例：

```text
event: message
data: {"agentName":"SupervisorPipeline","content":"分析问题","finalResponse":false,"partial":false,"type":"thinking"}

event: message
data: {"agentName":"SupervisorPipeline","content":"MathExpert","routeTarget":"MathExpert","finalResponse":false,"partial":false,"type":"route"}

event: message
data: {"agentName":"SupervisorPipeline","content":"阶段性说明...","finalResponse":false,"partial":true,"type":"reply"}

event: message
data: {"agentName":"SupervisorPipeline","content":"","finalResponse":false,"partial":false,"type":"reply"}

event: message
data: {"agentName":"SupervisorPipeline","content":"最终答案...","finalResponse":false,"partial":true,"type":"final"}

event: message
data: {"agentName":"SupervisorPipeline","content":"","finalResponse":true,"partial":false,"type":"final"}
```

## 兼容性与迁移提示

- 旧前端若只识别 `final`，需要补充 `thinking/route/reply` 的渲染逻辑
- 前端需要按 `partial` 拼接流式分片
- `finalResponse=true` 可作为本轮最终完成信号

## 已知约束

- Router 若输出非 JSON 或多余文本，可能导致路由解析失败
- 若 `sub-agents` 未包含可用 worker，supervisor workflow 会在启动时报错
- `max-iterations` 过小可能导致任务在未收敛时提前结束
