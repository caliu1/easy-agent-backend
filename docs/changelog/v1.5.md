# v1.5 更新日志

更新时间：2026-04-27

## 概览

v1.5 聚焦 DrawIO 生成链路稳定性与交互质量，核心目标是让多智能体行为标准与单智能体保持一致，并减少中间阶段语义漂移。

---

## 核心更新

### 1) 多智能体流程重构：自然语言直达 XML

原流程中包含 `RequirementAnalyst`、`DiagramArchitect` 等中间阶段，存在语义偏移与阶段冗余问题。v1.5 调整为：

- `SupervisorRouter -> DrawioXmlBuilder -> QualityReviewer`

关键变化：

- 移除 `RequirementAnalyst` 与 `DiagramArchitect`
- 不再走“自然语言 -> 布局 JSON -> XML”
- 改为“自然语言 -> XML -> 质检修复 -> 最终 XML”

---

### 2) 多智能体提示词优化

- 流程图判断节点必须为菱形（`decision/rhombus`）
- 判断分支必须标注 `Yes/No`（或 `Pass/Fail`）
- 连线必须正交，禁止斜线
- 锚点仅允许 `top/right/bottom/left`，禁止中心锚点
- 最终输出必须是完整 `mxfile` XML，禁止 markdown/JSON 包装/解释文本/片段输出

---

### 3) SupervisorRouter 能力增强

`SupervisorRouter` 不再只做路由，新增信息不足处理策略：

- 缺口识别（低歧义 / 高歧义）
- 低歧义缺口可默认补全并继续路由
- 高歧义缺口触发“最小澄清提问”，并同步给出默认补全方案
- 支持用户回复“按默认继续/你来补全”后直接进入生成链路

---

### 4) 阶段回复质量规则收敛

为解决“空话式中间态”问题，新增 route 回复质量约束：

- 每次 route 回复必须包含：
  - 依据（来自用户输入或补全假设）
  - 当前产出（至少 2 个事实细节）
  - 下一步动作
- 禁止“正在生成”“正在质检”“处理中”等空泛状态播报
