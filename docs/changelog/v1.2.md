# v1.2 更新日志

更新时间：2026-04-20

## 概览

v1.2 以“可运营的 Agent 平台能力”为目标，完成了从静态装配到动态装配、从单列表到广场分发、从手工配置到助手协同创建的升级。

本次版本可归纳为四条主线：

1. 后端配置中心化：Agent 配置入库并支持全生命周期管理。
2. 持久层纯 MP 化：使用 MyBatis-Plus 的 BaseMapper + Wrapper + Page 统一实现。
3. 前端产品化：新增“我的Agent / Agent广场”切换与创建页助手对话。
4. 流式体验优化：SSE 事件语义与前端增量渲染链路稳定化。

---

## 核心更新

### 1) Agent 动态装配（DB + YML 并存）

- 应用启动后优先加载数据库中已发布运行态配置。
- 同时继续加载 YML 系统 Agent，且支持与 DB 运行态共存。
- 当同 `agentId` 已存在激活运行槽位时，YML 系统 Agent 跳过激活，避免覆盖运行态。

关键文件：

- `easy-agent-app/src/main/java/cn/caliu/agent/config/AiAgentAutoConfig.java`

### 2) Agent 配置中心（CRUD + 生命周期）

新增管理接口能力：

- 创建：`POST /api/v1/agent_config_create`
- 更新：`POST /api/v1/agent_config_update`
- 删除：`POST /api/v1/agent_config_delete`
- 详情：`GET /api/v1/agent_config_detail`
- 列表：`GET /api/v1/agent_config_list`
- 我的列表：`GET /api/v1/agent_config_my_list`
- 广场列表：`GET /api/v1/agent_config_plaza_list`
- 分页检索：`POST /api/v1/agent_config_page_query`
- 发布运行：`POST /api/v1/agent_config_publish`
- 下线运行：`POST /api/v1/agent_config_offline`
- 回滚版本：`POST /api/v1/agent_config_rollback`
- 发布到广场：`POST /api/v1/agent_config_plaza_publish`
- 从广场下架：`POST /api/v1/agent_config_plaza_offline`

关键文件：

- `easy-agent-trigger/src/main/java/cn/caliu/agent/trigger/http/AgentConfigAdminController.java`
- `easy-agent-api/src/main/java/cn/caliu/agent/api/IAgentConfigAdminService.java`

### 3) 持久层“纯 MP 化”

- 去除依赖手写 upsert 的路径，统一走 MP BaseMapper + Wrapper。
- 条件检索与分页统一使用 MP `Page` 查询。
- 新增 MP 分页拦截器，数据库类型固定为 MySQL。

关键文件：

- `easy-agent-infrastructure/src/main/java/cn/caliu/agent/infrastructure/persistent/repository/AgentConfigRepository.java`
- `easy-agent-app/src/main/java/cn/caliu/agent/config/MybatisPlusConfig.java`

### 4) Agent 广场能力

新增广场相关领域字段与行为：

- `owner_user_id`：Agent 所属用户
- `source_type`：`USER` / `OFFICIAL`
- `plaza_status`：`ON` / `OFF`
- `plaza_publish_time`：发布时间

行为规则：

- “我的Agent”仅展示当前用户创建的用户 Agent。
- “Agent广场”展示已发布到广场且运行态可用的 Agent（可含官方 Agent）。
- 删除/下线时自动维护广场状态一致性。

SQL 脚本：

- `docs/dev-ops/mysql/sql/agent-config-v1.2-plaza.sql`

### 5) 前端升级（EasyAgent 平台）

首页能力升级：

- 顶部双 Tab：`我的Agent` 与 `Agent广场` 切换展示。
- Agent 卡片管理菜单：发布运行、下线运行、回滚、发布广场、广场下架、删除、快速更新。
- 删除成功后即时从 UI 列表移除，并使用浏览器通知提示（自动消失）。

创建页升级：

- 新建页支持填写 `model`、`completions-path`、`embeddings-path`。
- 支持 `agents`、`agentWorkflows` 多项动态增删并由前端拼装合法 `configJson`。
- 右侧固定对话助手（`AgentConfigWriterAgent`）流式输出并动态修复截断 JSON，实时回填左侧表单。

关键文件：

- `agent-scaffold-draw-io-front/src/app/page.tsx`
- `agent-scaffold-draw-io-front/src/app/agent-create/page.tsx`
- `agent-scaffold-draw-io-front/src/api/agent.ts`

### 6) 流式事件链路增强

后端持续输出标准事件：

- `thinking`
- `route`
- `reply`
- `final`

前端按 SSE block 增量解析并分片渲染，保障“思考过程 + 主回复”在时序上的稳定展示。

关键文件：

- `easy-agent-trigger/src/main/java/cn/caliu/agent/trigger/http/AgentServiceController.java`
- `agent-scaffold-draw-io-front/src/api/agent.ts`

## 升级后快速验证

### 后端

1. 启动应用并确认日志中出现“DB runtime loaded”与“YML system config loaded”相关信息。
2. 调用 `agent_config_create` 创建一个用户 Agent。
3. 调用 `agent_config_publish` 发布运行。
4. 调用 `agent_config_plaza_publish` 发布到广场。
5. 分别调用 `agent_config_my_list` 与 `agent_config_plaza_list` 验证隔离展示。

### 前端

1. 登录后进入首页，确认可在“我的Agent / Agent广场”间切换。
2. 在新建页与 `AgentConfigWriterAgent` 对话，观察表单是否随流式输出动态更新。
3. 删除 Agent 后，确认卡片立即消失且浏览器通知提示“已删除Agent：xxx”。

## 兼容性说明

- 旧前端若未接入广场相关字段（`ownerUserId/sourceType/plazaStatus`），需要补齐 DTO 映射。
- 若已有同 `agentId` 的 DB 激活运行槽位，YML 同 ID 系统 Agent 不会覆盖激活态（符合预期）。
- 本版本默认数据库为 MySQL，并启用 MP MySQL 分页拦截器。
